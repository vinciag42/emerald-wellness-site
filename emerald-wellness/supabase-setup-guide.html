Step 1
Create your Supabase project
Free account — no credit card needed for the free tier which handles up to 50,000 users.
  1. Go to supabase.com → click Start your project
  2. Sign up with GitHub (recommended) or email
  3. Click New project
  4. Name it emerald-wellness
  5. Choose a Database Password — save it securely
  6. Select region closest to your users (e.g. US East for Columbus, OH)
  7. Click Create new project — takes ~2 minutes to provision
Step 2
Run the database schema
Creates all tables, indexes, RLS policies, and triggers in one click.
  1. In your Supabase project → left sidebar → click SQL Editor
  2. Click New query
  3. Open supabase-schema.sql (included in your project files)
  4. Copy the entire contents and paste into the SQL Editor
  5. Click Run (or press Cmd+Enter)
  6. You should see: Success. No rows returned.
✦ Verify it worked
Go to Table Editor in the left sidebar. You should see: profiles, health_goals, referrals, waitlist, email_log, plans — all created.
Step 3
Get your API keys
Two values you'll paste into signup.html — your project URL and anon key.
  1. In your Supabase project → left sidebar → Project Settings (gear icon)
  2. Click API in the settings menu
  3. Copy your Project URL — looks like:
    https://abcdefghijklmnop.supabase.co
  4. Copy your anon public key (the long JWT string under "Project API keys")
⚠ Key safety
The anon key is safe to use in frontend JavaScript — it's designed to be public. Never use your service_role key in frontend code — it bypasses all RLS security.
Step 4
Add your keys to signup.html
Open signup.html and find the Supabase config block near the top. Replace the two placeholder values.
signup.html — Supabase config block
// Find this block near the top of signup.html <head> // Replace BOTH values with your actual keys from Step 3 const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co'; const SUPABASE_ANON = 'YOUR_ANON_PUBLIC_KEY'; // Replace with your real values, e.g.: const SUPABASE_URL = 'https://abcdefghijklmnop.supabase.co'; const SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
Step 5
Configure Supabase Auth settings
Required for email confirmation and redirect to work correctly.
  1. In Supabase → Authentication → URL Configuration
  2. Set Site URL to: https://emeraldwellness.health
  3. Add to Redirect URLs:
    https://emeraldwellness.health/signup
    http://localhost:3000/signup (for local testing)
  4. In Authentication → Email Templates → customize the confirmation email with your Emerald Wellness branding (optional but recommended)
  5. In Authentication → Providers → enable Email (already on by default)
Step 6
Test your first signup
Verify data flows all the way into Supabase before going live.
  1. Open signup.html in your browser
  2. Complete all 3 steps with a real email address you own
  3. Click Start My Free 7-Day Trial
  4. In Supabase → Authentication → Users — you should see your new user
  5. In Supabase → Table Editor → profiles — your full profile row should be there
  6. Check health_goals table — your selected goals should be saved as individual rows
  7. Check your email — you should receive a Supabase confirmation email
✦ Browser console check
Open DevTools (F12) → Console while signing up. You should see no errors. If you see Profile insert error in the console, your RLS policy may need adjustment — go to Supabase → Authentication → Policies and ensure the "Service role can insert profiles" policy is enabled.
