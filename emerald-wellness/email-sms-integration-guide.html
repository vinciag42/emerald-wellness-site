⚡ Compliance Note
TCPA (US) and CTIA guidelines require: explicit opt-in consent before sending SMS, sender identification in every message, and a clear STOP opt-out mechanism. The sign-up form already includes an SMS consent checkbox — ensure it is checked before triggering the SMS flow. Keep all opt-out records for a minimum of 5 years.

Add this to your signup.html submitSignup() function — after validation passes and before showing the success state:

// Add inside submitSignup() after validation, before goStep(4) // 1. Collect form data const signupData = { first_name: document.getElementById('su-first').value.trim(), last_name: document.getElementById('su-last').value.trim(), email: document.getElementById('su-email').value.trim(), plan: selectedPlan, referral: document.getElementById('su-ref').value.trim() || null, sms_consent: document.getElementById('su-sms-consent')?.checked || false, goals: Array.from(document.querySelectorAll('.goal-btn.selected')) .map(b => b.textContent.trim()), referral_link: 'emeraldwellness.health/ref/EW' + Math.random().toString(36).substr(2,6).toUpperCase() }; // 2. Send to your backend / ESP webhook fetch('https://your-backend.com/api/signup', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(signupData) }) .then(res => res.json()) .then(data => { console.log('Signup sent to ESP', data); }) .catch(err => { // Non-blocking — proceed to success even if API fails console.error('ESP trigger error:', err); }); // 3. Show success (always runs regardless of API response) goStep(4);

Klaviyo Setup (Recommended)

Step-by-step

  1. Create a Klaviyo account at klaviyo.com and get your Public API Key from Account → Settings → API Keys.
  2. Create a new List called "Emerald Wellness Members" and note the List ID.
  3. In your backend API route, use the Klaviyo v3 API to add the subscriber and trigger the welcome flow:
    // Node.js backend example const KLAVIYO_KEY = process.env.KLAVIYO_PRIVATE_KEY; // Add profile to Klaviyo await fetch('https://a.klaviyo.com/api/profiles/', { method: 'POST', headers: { 'Authorization': `Klaviyo-API-Key ${KLAVIYO_KEY}`, 'Content-Type': 'application/json', 'revision': '2024-02-15' }, body: JSON.stringify({ data: { type: 'profile', attributes: { email: signupData.email, first_name: signupData.first_name, last_name: signupData.last_name, properties: { plan: signupData.plan, referral_link: signupData.referral_link, goals: signupData.goals.join(', '), sms_consent: signupData.sms_consent } } } }) }); // Fire "Signup" event to trigger welcome flow await fetch('https://a.klaviyo.com/api/events/', { method: 'POST', headers: { ... }, body: JSON.stringify({ data: { type: 'event', attributes: { metric: { data: { type: 'metric', attributes: { name: 'Signup' }}}, profile: { data: { type: 'profile', attributes: { email: signupData.email }}}, properties: { plan: signupData.plan } } } }) });
  4. In Klaviyo → Flows → Create Flow → "Metric Triggered" → choose the Signup event.
  5. Add an Email action with 0-minute delay. Paste the HTML from email-welcome-template.html. Map merge tags: {{FIRST_NAME}} → {{ first_name }}.
  6. If SMS consent = true, add an SMS action with 30-second delay. Paste the SMS copy above.
  7. Activate the flow. Test with your own email address.

Phone number collection
The current signup.html form collects email only. To enable SMS, add a phone number field to Step 3 of the sign-up form with a separate SMS consent checkbox: "Send me a welcome text message (optional)." Never send SMS without explicit opt-in.

No-Code Option: Zapier

Setup in 15 minutes — no backend required

  1. Add a Webhook trigger to signup.html — in the submitSignup() function, add a fetch POST to your Zapier Webhook URL (from Zapier → Trigger → Webhooks by Zapier → Catch Hook).
  2. Zapier Zap 1 — Email: Trigger: Webhook → Action: Mailchimp "Add/Update Subscriber" → Action: Mailchimp "Send Transactional Email" using the welcome template.
  3. Zapier Zap 2 — SMS: Trigger: Same Webhook → Filter: only if sms_consent == true → Action: Twilio "Send SMS Message" with the SMS copy above.
  4. Test both Zaps. Turn them on. Done.

✅ Pre-Launch Testing Checklist

  • Sign up with a test email → confirm welcome email arrives within 60 seconds
  • Check email renders correctly in Gmail, Apple Mail, and Outlook (use Litmus or Email on Acid)
  • Test SMS delivery with a real phone number
  • Test STOP opt-out — reply STOP and confirm no further SMS are sent
  • Test unsubscribe link in email footer
  • Verify {{FIRST_NAME}} and all merge tags are populated (not shown as raw variables)
  • Confirm referral link is unique per user
  • Test across all 4 plans — confirm {{PLAN_NAME}}, {{PLAN_PRICE}}, {{FIRST_MONTH_PRICE}} populate correctly
  • Verify CAN-SPAM compliance: sender name, physical address, unsubscribe link all present
  • Check spam score (use mail-tester.com) — aim for 9+/10
